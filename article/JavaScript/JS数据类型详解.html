# typeof

鉴于js的数据类型是一种松散类型，即需要一种检测变量数据类型的手段——typeof

typeof可能返回下列字符串

  • "undefined" 变量未定义
  • "boolean" 如果该变量是一个布尔值
  • "string" 如果该变量是个字符串
  • "number" 数字
  • "object" 这个值是对象或者是null
  • "function" 该变量是个函数

注意

typeof null返回object：特殊变量null被认为是一个空的对象引用
typeof 正则表达式 返回object
typeof 是一个操作符而不是函数，所以typeof a或者 typeof(a)都行

以下都是合法的Number类型

(1+2)*5/2     // 3.5

# Number的进制表示

除了十进制，整数还可以通过八进制或十六进制表示。

  • 八进制(0~8) 第一位为零 例：078
  • 十六进制(0 ~9,A ~F) 前两位为0X表示 例：0xA
在进行算数计算时：所有八进制和十六进制将转化为十进制数值

# Number的浮点数

浮点数：即为数值中必须包含一个小数点，且小数点后面必须有一位数字

保存浮点数需要的空间是保存整数的两倍，所以js会把10.0这种浮点数转化为整数

# NaN

not a number
用于表示一个本来返回数字的操作未返回数值的情况(这样就不会抛出错误了)

  • 任何涉及到NaN的操作都返回NaN
  • NaN和任何值都不相等，包括NaN本身

isNaN()函数：该函数接受一个参数(该参数可以是任何类型),并判断这个数是否“数值”。
isNaN()在接受一个值后会尝试将这个值转换成数值，任何不能转换成数字的值将会返回false

isNaN("10")     // "10"->10 true
isNaN(true)     // true->1 true
isNaN("bule")   // false 不可转变为数值

# 数值转换

有三个函数可以把非数值，转换为数值
Number()
parseInt()
parseFloat()

# Number
  • boolean值直接转换成1或0
  • 数字值原样返回
  • null返回0
  • undefined返回NaN
  • 对象，调用ValueOf(),在根据上述规则处理，如果是NaN则调用toString(),再根据上述的规则处理返回的字符串数值。
  • 字符串
    • 只包含数字(带正负号)，转换为十进制，忽略前导零。（011->11）
    • 包含有效的十六进制(0xf),转换为相同大小的十进制
    • 字符串为空返回0
    • 包含上述之外的字符，返回NaN
Number(true)       // 1
Number(null)       // 0
Number(undefined)  // NaN
Number("0011")     // 11
Number("")         // 0
NUmber("0xA")      // 10
Number("true")     // NaN
# parseInt()

由于Number()函数在转换字符串时的复杂不合理，处理字符串更常用parseInt()

  • 会忽略字符串前面的空格，直到找到第一个非空字符串
  • 如果第一个字符串不是数字或负号，返回NaN（parseInt("")为NaN，而Number是返回0）
  • parseInt()从第一个数字字符串开始解析，直到碰到一个非数字字符串
    • 例："1234blue" -> 1234 "11.5" -> 11
  • parseInt()可以识别各种进制，并转化成十进制
parseInt("1234blue")    // 1234
parseInt("")            // NaN
parseInt("blue")        // NaN
parseInt("0xA")         // 10 十六进制
parseInt("22.5")         // 22
paseInt("70")            // 70 忽略前导 0
// ES5 中070不被识别为八进制，忽略前导零（ES3中识别）
parseInt("070",8)       // 56 可以指定进制
parseInt("AF", 16)      // 175 甚至可以忽略进制的标志符 0x
# parseFloat()
  • 和parseInt()类似
  • 第一个小数点有效 "10.1.1 -> 10.11"
  • 只解析十进制，十六进制会被解析为0
  • 没有第二个参数
parseFloat("1234bule")     // 1234
parseFloat("")             // NaN
parseFloat("false")        // NaN
parseFloat("23.2.2")       // 23.22
parseFloat("3.15e3")       // 3250
parseFloat("0xA")          // 0

# toString()

number，bolean，object，string都有个toString方法。该方法返回字符串的一个副本，但null和undefined没有这个方法

let num = 10;
num.toString()     // "10"
num.toString(2)     // "1010" 可以写一个参数，确定返回的进制格式。
# String()

适用于所有情况，能将任何类型的值转换为字符串

转换规则

  • 如果该值有toString方法，则调用该方法并返回相应的结果
  • 如果值是null，则返回 "null"
  • 如果值是undefined,则返回 "undefined"
let num1 = 10;
let num2 = true;
let num3 = null;
let num4 ;
alert(String(num1))     // "10"
alert(String(num2))     // "true"
alert(String(num3))     // "null"
alert(String(num4))     // "undefined"

# Object

对象就是一组数据和功能的集合

对象的创建

let o = new Object()    // 没有参数，括号可以省略

Object类型是所有实例的基础
Object的每个实例都具有下列的属性和方法

  • constructor  保留着用于创建当前对象的函数。上例而言，构造函数(constructor)就是Object()

  • hasOwnProperty(propertyName)  用于检查给定的属性在当前的对象实例中是否存在

    • 该方法的参数propertyName必须以字符串形式。例：o.hasOwnProperty(property("name"))

  • isPrototypeOf(Object) 用于检查传入的对象是否是当前对象的原型

  • propertyIsEnumberable(propertyName)  用于检查给定的属性是否能使用for-in来枚举。参数同样需以字符串

  • toLocaleString()  返回对象的字符串表示，该字符串与执行环境的地区对应

  • toString()  返回对象的字符串表示

  • valueOf 返回对象的字符串，数值或布尔值表示。通常与toString()方法的返回值相同

# 变量

# 前言

js中的数据类型可以分为两种，基本数据类型和引用数据类型。

# 变量赋值

基本数据类型和引用数据类型，在js中的储存方式不同。

# 基本数据类型

number, boolean, string, null, undefined

基本数据类型的变量储存在栈内存中，对于基本数据类型的赋值，相当于复制一个副本

# 引用数据类型

Object, Array, Function, RegExp

引用数据在栈内存中存储的是指针，指向堆内存中的对象。复制也只是复制了一个指针，指向的还是一个对象。

# 解决办法

浅拷贝和深拷贝

# 传递参数

js中所有函数的参数都是按值传递的
Function的参数储存在一个名为arguments的类数组中，通过arguments[0]调用

在js中函数参数的传递，相当于赋值传递。将变量赋值给函数内部的局部变量，局部变量再去参与函数内部的运算

function addTen(num) {
    num += 10;
    return num
}
let a = 10;
let result = addTen(a)

    近似于↓
function addTen(num) {
    num = a;
    num += 10;
    return num;
}
// 将a值赋值给num，num参与运算

# 注意

在传递参数中，基本数据类型和引用数据类型有区别

引用数据类型对参数的操作会影响原变量

function setName(obj) {
    obj.name = "xu"
}
let person = new Object();
setName(person);
console.log(person.name)    // "xu"
